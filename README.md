- Windows 10 or Windows 11
- 64-bit PC recommended

## 🧭 How to Use

If you are using more than one DLL, load them in the order you want them to run.

## 🔍 Choosing the Right Process

When you pick a process, match it to the app you want to work with.

- Use the process name shown in the list
- Do not guess if two apps have similar names
- Check whether the target app is 32-bit or 64-bit
- Use the same architecture for the DLL and target app

If the target app closes or does not respond, pick another process and try again.

## ⚙️ Injection Modes

### Standard Inject
This is the basic mode. It is simple and works for many cases.

### Manual Map
This mode loads the DLL in a different way. It is used when normal injection does not work.

### Stealth Mode
This mode hides parts of the inject process. It is used in advanced setups.

### Multi DLL
This lets you load more than one DLL in the same session.

## 🧰 Troubleshooting

### The app does not open
- Right-click the file and run it as administrator
- Make sure the file was fully extracted
- Check that Windows did not block the file

### The DLL does not load
- Check that the DLL matches the target app type
- Try a different injection mode
- Make sure the target process is running
- Confirm the DLL file path is correct

### The target app closes
- Match x86 with x86 and x64 with x64
- Try another process
- Use a clean DLL build
- Start the injector after the target app is open

### Windows shows a security prompt
- Allow the app if you trust the source
- Keep the release files in one folder
- Run the app with the permissions it needs
